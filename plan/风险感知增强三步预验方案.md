# 风险感知增强三步预验方案（给 Codex / Agent 使用）

## 0. 当前问题

当前两组 seed 预验结果显示：

- `Ours` 在 `Fee` 和打包能力上较强；
- `Ours` 在 `Fairness` 上基本不弱；
- 但 `Ours` 的 `risk_exposure`、`top10_risk / edge10_risk`、`feasible_rate` 不如 `Dynamic Tri-Objective`；
- 说明当前 PPO 的 operating point 偏向收益 / 打包效率，而不是风险感知均衡。

因此，当前不需要大改模型结构，只需要做：

> **风险惩罚与约束权重增强的配置级预验。**

# 3. 三组风险增强配置

## 3.1 配置 A：轻度风险增强

### 目标

尽量保留收益优势，同时观察风险指标是否下降。

### 建议参数

```text
GAMMA_RISK = 0.8
TERMINAL_RISK_EXPOSURE_WEIGHT = 0.35
TERMINAL_TOP10_RISK_WEIGHT = 0.25
RISK_ADJUSTED_FEE_LAMBDA = 0.5
```

如果代码已有 `edge10` 终止风险项，则同时设置：

```text
TERMINAL_EDGE10_RISK_WEIGHT = 0.35
```

### 预期

- `Fee` 小幅下降；
- `risk_exposure` 有一定下降；
- `top10_risk / edge10_risk` 有一定改善；
- `packing_ratio` 基本保持；
- `feasible_rate` 略有提升。

---

## 3.2 配置 B：均衡风险增强【优先推荐】

### 目标

将 PPO 从收益优先 operating point 推向风险感知均衡 operating point。

### 建议参数

```text
GAMMA_RISK = 1.0
TERMINAL_RISK_EXPOSURE_WEIGHT = 0.5
TERMINAL_TOP10_RISK_WEIGHT = 0.4
RISK_ADJUSTED_FEE_LAMBDA = 0.6
```

如果代码已有 `edge10` 终止风险项，则同时设置：

```text
TERMINAL_EDGE10_RISK_WEIGHT = 0.5
```

### 预期

- `Fee` 允许小幅下降；
- `risk_exposure` 明显下降；
- `top10_risk / edge10_risk` 明显下降；
- `feasible_rate` 提升；
- `trade_score` 或 `composite_score` 缩小与 `Dynamic Tri-Objective` 的差距；
- 更适合作为正式 5 seed 候选配置。

---

## 3.3 配置 C：强风险约束

### 目标

作为风险优先 operating point 观察，不一定作为正式主模型。

### 建议参数

```text
GAMMA_RISK = 1.2
TERMINAL_RISK_EXPOSURE_WEIGHT = 0.7
TERMINAL_TOP10_RISK_WEIGHT = 0.5
RISK_ADJUSTED_FEE_LAMBDA = 0.8
```

如果代码已有 `edge10` 终止风险项，则同时设置：

```text
TERMINAL_EDGE10_RISK_WEIGHT = 0.7
```

### 预期

- `risk_exposure` 下降明显；
- `top10_risk / edge10_risk` 下降明显；
- `Fee` 和 `packing_ratio` 可能下降；
- 可作为风险优先策略参考，不一定作为正式结果。

# 4. 每组预验需要输出的关键指标

每组训练完成后，比较 `Ours` 与 `Dynamic Tri-Objective`。

必须输出以下指标：

```text
block_fee_mean
fairness_mean
old_tx_pack_rate_mean
risk_exposure_mean
top10_risk_mean
edge10_risk_mean
feasible_rate
packing_ratio_mean
trade_score_mean
risk_aware_trade_score_mean
constrained_trade_score_mean
```

如果代码支持，还必须输出：

```text
risky_inclusion_rate_mean
```

用途：

> 判断模型是否只是通过少打包高风险交易来伪造低风险表现。

---

# 5. 判断标准

不要要求 `Ours` 在所有指标上第一。

成功标准是：

```text
risk_exposure 从当前约 0.21 明显下降，目标区间 0.15~0.17；
top10_risk / edge10_risk 明显下降；
feasible_rate 从当前约 0.65 提升，目标至少接近或超过 0.70；
block_fee 不明显低于 Dynamic Tri-Objective；
fairness 维持在 0.90 以上；
old_tx_pack_rate 接近 1.0；
packing_ratio 不大幅下降；
risky_inclusion_rate 不明显下降。
```

如果配置 B 达到以上条件，优先选择配置 B 作为后续正式 5 seed 候选配置。
