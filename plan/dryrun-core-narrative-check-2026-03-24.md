# Dryrun 数据与论文核心叙事一致性核查（2026-03-24）

## 结论

**总体判断：基本满足论文核心叙事，但必须使用“多目标折中领先”而非“每个指标都最优”的表述。**

- 可成立的主叙事：在主场景中，`ours` 在收益（`block_fee_mean`）和综合分数（`composite_score_mean`）上领先，同时维持较高公平性。
- 不可成立的强叙事：`ours` 不是公平性最优，也不是风险最优；若写成“全面最优”会与数据冲突。
- 证据等级限制：当前 run 仅单种子（`seeds=[42]`），结论应标注为 dryrun 级、探索性证据。

## 1) 主实验是否支持“多目标折中领先”

从 `aggregated_main.json` 可读出：

- `ours.block_fee_mean = 1682.94`，高于 `gas = 1598.61`、`fifo = 1543.27`。
- `ours.composite_score_mean = 0.6122`，在各方法中最高。
- `ours.fairness_mean = 0.9085`，虽高但低于 `fifo = 0.9268`。
- `ours.risk_exposure_mean = 0.2071`，优于 `gas = 0.3417`，但劣于 `fee_risk_linear = 0.0774` 和 `heuristic = 0.1047`。

**解释**：
这组结果支持“相较高收益贪心基线，本文方法在风险-公平-收益之间取得更好的工程折中”；
但不支持“公平/风险绝对最优”叙事。

---

## 2) 约束叙事是否成立

从 `constrained_eval_summary.json` 看：

- `ours.feasible_rate = 0.65625`，低于 `fifo = 0.8125`。
- `ours.constrained_fee_mean = 0.6699`，高于 `fifo = 0.6353`。
- 排名 `ranking` 为 `fifo` 在前、`ours` 第二。

**解释**：
约束实验更像“`ours` 在可行样本上的收益更高，但可行率不占优”。
因此论文不能写成“约束条件下 ours 全面第一”，应写成“收益-可行率之间存在权衡”。

## 4) 可直接用于论文正文的安全表述

建议使用如下口径：

1. 本文方法在主场景中实现最高收益与最高综合指标，体现出较强工程实用性。
2. 相比高收益贪心方法，本文方法显著降低风险暴露并保持高公平水平。
3. 本文方法并不追求全局最小风险或绝对最高公平，而是追求风险-公平-收益的可控折中。
4. 由于当前为单种子 dryrun，统计结论需在多种子正式实验中复核。

## 5) 回答你的关键问题：只改论文措辞是否足够？是否无需改实验代码？

短答案：**只改措辞可以让“当前 dryrun 结果”与论文文字不冲突，但不足以支撑正式投稿/答辩级证据强度。**

### 5.1 什么时候“只改措辞”是够的

若你的目标是：

- 先完成阶段性稿件；
- 明确把本轮结果定义为 **single-seed dryrun exploratory evidence**；
- 在正文中避免“全维最优/约束第一”这类绝对化表述；

那么目前**可以先不改训练与推理代码**，通过修正文案实现叙事一致。

### 5.2 什么时候“只改措辞”不够

若你的目标是：

- 正式论文定稿（尤其是审稿人会追问统计显著性与泛化）；
- 主张“方法稳定优于基线”；
- 主张“约束场景下方法整体更优”；

则仅改文字不够。你至少需要补**实验层**而不一定先改**算法代码层**。

### 5.3 建议的最小增量路线（按优先级）

1. **先补多种子复现实验（高优先级，低代码风险）**  
   目标：把 `dryrun_single_seed` 提升到可统计报告的证据等级。
2. **固定主叙事口径并同步表格结构（高优先级）**  
   主表同时给出 fee/fairness/risk/composite，避免选择性呈现质疑。
3. **仅在多种子后仍出现系统性冲突时，再做代码改动（中优先级）**  
   例如若 `feasible_rate` 持续落后，可再考虑 reward 权重或约束策略微调。

### 5.4 实操建议（你现在就可以执行）

- 论文文本立即改：把“全面最优”改为“多目标折中优势”。
- 实验计划立即补：先跑 3~5 seeds 主实验 + 关键鲁棒性。
- 代码暂缓改：等多种子结果确认“稳定短板”后再定向改，避免过早优化。

**最终建议**：
> 你现在可以先不改实验代码，但不建议“只改措辞就结束”。更稳妥做法是“先改措辞 + 补多种子实验”；代码改动作为第二阶段、由证据驱动。
