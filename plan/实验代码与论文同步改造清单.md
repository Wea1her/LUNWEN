# 2. P0-1：论文奖励函数必须与代码实现对齐

## 2.1 当前问题

论文中目前仍然使用简单四项奖励：

\[
r_t = \alpha r_t^{fee} + \beta \Delta r_t^{fair} - \gamma r_t^{risk} + \eta r_t^{valid}
\]

但代码中的实际奖励已经不是简单四项，而是包含更细的分解项，包括：

- 手续费收益；
- age-aware 等待奖励；
- oldest-cover 长等待交易覆盖奖励；
- 位置敏感风险惩罚；
- packing 激励；
- late-high-fee recovery 激励；
- terminal fairness；
- starvation penalty；
- terminal risk exposure；
- terminal top10/edge risk；
- risky rank 偏离惩罚；
- unused gas penalty；
- STOP 相关惩罚。

因此，论文如果继续写简单四项奖励，会导致“方法公式”和“代码实现”不一致。

## 2.2 建议改造方式

建议保留“三目标主线”，但把奖励函数写成层次化形式：

\[
r_t = r_t^{step} + r_t^{terminal} + r_t^{valid}
\]

其中：

\[
r_t^{step}
=
\alpha r_t^{fee}
+
\beta_{age} r_t^{age}
+
\beta_{old} r_t^{oldest}
-
\gamma r_t^{risk}
+
\omega_p r_t^{packing}
+
\omega_l r_t^{latefee}
\]

终止奖励写成：

\[
r_T^{terminal}
=
\beta_T r_T^{fair}
-
\gamma_s r_T^{starvation}
-
\gamma_e r_T^{edge-risk}
-
\omega_g r_T^{unused-gas}
\]

有效性奖励写成：

\[
r_t^{valid}
=
r_t^{invalid}
+
r_t^{stop}
\]

## 2.3 论文中推荐解释

可以在论文中写：

> 代码实现中，本文将收益—等待公平—风险暴露三类目标进一步细化为单步奖励、终止奖励和有效性约束三部分。单步奖励用于引导策略在逐笔选择过程中兼顾手续费收益、长等待交易服务和风险位置控制；终止奖励用于约束最终区块序列的等待公平、饥饿比例、敏感位置风险暴露和 Gas 利用率；有效性奖励用于惩罚非法动作和过早 STOP。该分解仍服务于收益、等待公平和位置敏感风险暴露三目标。

## 2.4 修改位置

需要同步修改：

```text
chapters/journal/03-problem-modeling.tex
chapters/journal/04-method.tex
chapters/journal/05-experiments.tex
```

如果正文篇幅有限，可在正文写主公式，在附录给出详细奖励项。

# 4. P0-3：Dynamic Tri-Objective Greedy 需要重新实现

## 4.1 当前问题

当前代码中已有 `center_aware` 或类似的强基线入口，但当前实现更像：

> 先按收益、等待等静态分数排序，再把高风险交易插入中部。

它不是严格的 V5 中定义的逐步动态三目标贪心：

\[
s_t(tx_i)
=
\lambda_f f_i
+
\lambda_w w_i
-
\lambda_r r_i\phi(\rho_t)
\]

其中 \(\rho_t\) 是当前序列位置，\(\phi(\rho_t)\) 是位置风险权重。

## 4.2 需要新增的强基线

建议新增正式函数：

```python
dynamic_tri_objective_greedy_sort(pool, config, lambdas)
```

核心逻辑：

```python
selected = []
remaining = list(pool)
gas_left = block_gas_limit

while True:
    valid = get_valid_candidates(remaining, selected, gas_left)
    if not valid:
        break

    rho = len(selected) / estimated_block_capacity
    pos_weight = phi(rho)

    for tx in valid:
        score = (
            lambda_f * fee_norm(tx)
            + lambda_w * wait_norm(tx)
            - lambda_r * risk_score(tx) * pos_weight
        )

    chosen = argmax(score)
    selected.append(chosen)
    remaining.remove(chosen)
    gas_left -= chosen.gas
```

## 4.3 该基线必须满足

- 每一步重新计算当前位置 \(\rho_t\)；
- 每一步只在合法交易集合中选择；
- 使用相同 Gas 约束；
- 使用相同 Nonce 依赖约束；
- 使用与 PPO 相同的位置风险函数；
- 使用与 PPO 相同的特征归一化；
- 支持 STOP 或自然终止；
- 在验证池上调参；
- 在测试池上只评估一次。

## 4.4 当前 center-aware 基线建议改名

当前已有实现可保留，但建议改名为：

```text
Center-Insertion Heuristic
```

而真正的新增基线命名为：

```text
Dynamic Tri-Objective Greedy
```

这样避免“名称很强，但实现不是动态三目标”的问题。

```text

experiments/latex_tables.py
```

# 6. P0-5：正式统计应以 seed 级为主

## 6.1 当前问题

代码中已经有 seed-level summary，这是好的。  
但正式显著性检验仍可能使用 episode-level paired test。

问题是：

> 1000 个测试 episode 可以降低评估均值方差，但不能等价于 1000 次独立训练。

如果直接用 5000 个 episode 做显著性检验，会低估训练随机性，容易得到伪显著性。

## 6.2 推荐正式统计流程

每个 seed 先得到一个均值：

```text
seed_42: PPO mean metric, baseline mean metric
seed_123: PPO mean metric, baseline mean metric
seed_456: PPO mean metric, baseline mean metric
seed_789: PPO mean metric, baseline mean metric
seed_2025: PPO mean metric, baseline mean metric
```

然后对这 5 个配对值做统计。

## 6.3 需要新增的统计函数

建议新增：

```python
run_seed_level_paired_tests(seed_level_statistics)
```

输出：

```text
seed_level_paired_tests.json
table_seed_level_significance.tex
```

统计内容：

- seed-level mean difference；
- paired t-test；
- Cohen's \(d_z\)；
- bootstrap 95% CI；
- Holm correction；
- ours_better_seed_rate。

## 6.4 episode-level 检验的定位

episode-level paired test 可以保留，但应降级为：

> 诊断性分析 / 附录分析。

正文正式显著性结论应以 seed-level 统计为主。

## 6.5 论文表述建议

可以写：

> 本文以独立训练 seed 作为主要统计单位。每个 seed 的策略在固定共享测试池上评估后，先对测试 episode 求均值，再对 5 个 seed 进行跨 seed 汇总和配对统计。Episode 级配对检验仅作为附录中的诊断性分析。

# 15. 最终判断

当前代码已经可以支撑小论文主体，但不能直接作为严格 V5 最终版提交。

最关键的问题不是“实验不够多”，而是：

> 代码实现已经比论文草稿复杂，论文必须同步升级；同时，强基线、统计和诊断指标还需要补强。

完成本清单中的 P0 项后，代码即可较稳地支撑小论文。  
完成 P1 项后，实验方案会更加接近 V5 正式送审版本。  
P2 项可以作为毕业论文或后续扩展工作。
