# 综合折中函数版本：正式实验前修改清单

## 0. 总体结论

本次加入 `trade_score`、`risk_aware_trade_score` 和 `constrained_trade_score` 的方向是正确的，能够支撑论文中“给定目标偏好下的多目标折中最优”叙事。

但在开始正式 5 seed 实验前，建议完成以下小修正：

1. 统一风险指标口径：从 `top10_risk` 统一到 `edge10_risk`；
2. 在论文实验章节正式定义 \(S_{\text{trade}}\)；
3. 明确 `trade_score` 是“评价指标”还是“选模指标”；
4. 降级旧的 `composite_score` 叙事，避免多个综合指标互相冲突；
5. 规范 `RiskRef` 和 `EdgeRef` 的来源；
6. 跑一次单 seed sanity check 后再启动正式 5 seed 实验。

这些修改不属于大改实验框架，而是为了让“综合折中函数”真正成为可解释、可复现、可防守的论文评价指标。

---

## 1. 修改项一：统一风险指标口径

### 1.1 当前问题

论文叙事强调的是：

> 高风险交易在区块头部和尾部等敏感位置的风险暴露。

因此，正式风险指标应优先使用：

```text
edge10_risk
```

而不是只看区块头部的：

```text
top10_risk
```

当前部分代码仍然使用 `top10_risk` 参与 narrative guard 或 checkpoint 评价，这会造成口径不一致：

- 论文说“头尾敏感区”；
- 代码部分地方只看“头部 10%”。

### 1.2 建议修改

将正式风险口径统一为：

```text
edge10_risk
```

即：

> 区块前 10% 与后 10% 敏感区域中的高风险交易占比。

### 1.3 需要检查/修改的位置

重点检查以下文件：

```text
experiments/run_experiments.py
experiments/train.py
experiments/metrics.py
experiments/latex_tables.py
```

重点搜索：

```bash
grep -R "top10_risk" -n experiments/
grep -R "edge10_risk" -n experiments/
```

### 1.4 推荐修改方式

如果某处用于“正式风险判断”，将：

```python
("top10_risk_mean", False)
```

改为：

```python
("edge10_risk_mean", False)
```

如果希望保留头部风险诊断，可以同时保留：

```python
("risk_exposure_mean", False),
("edge10_risk_mean", False),
("top10_risk_mean", False),  # diagnostic only
```

但论文主叙事中应优先解释：

```text
risk_exposure
edge10_risk
```

### 1.5 论文表述

建议写：

> 本文使用 `risk_exposure` 衡量总体位置敏感风险暴露，并使用 `edge10_risk` 衡量区块头部与尾部敏感区域中的高风险交易占比。`top10_risk` 仅作为补充诊断指标，不作为主要风险结论依据。

## 2. 修改项二：论文中正式定义多目标折中函数 \(S_{\text{trade}}\)

### 2.1 当前问题

代码中已经实现：

```text
trade_score
risk_aware_trade_score
constrained_trade_score
```

但论文实验章节还没有正式定义这些指标，容易造成：

- 代码有指标，论文没有解释；
- 审稿人不知道综合分如何计算；
- “折中最优”缺少数学定义。

### 2.2 建议新增章节

在论文实验章节“评价指标”之后，新增小节：

```latex
\subsubsection{多目标折中得分}
```

### 2.3 推荐写入内容

```latex
为避免仅依据单一指标评价排序策略，本文进一步定义多目标折中得分
$S_{\text{trade}}$，用于衡量排序结果在手续费收益、等待公平、系统效率
和位置敏感风险控制之间的综合表现。具体定义如下：

\[
S_{\text{trade}}
=
w_f \tilde{F}
+
w_j \tilde{J}
+
w_o \tilde{O}
+
w_g \tilde{G}
+
w_p \tilde{P}
+
w_r \tilde{R}_{safe}
+
w_e \tilde{E}_{safe}.
\]
```

继续写：

```latex
其中，$\tilde{F}$ 表示归一化手续费收益，$\tilde{J}$ 表示等待时间公平性，
$\tilde{O}$ 表示长等待交易打包率，$\tilde{G}$ 表示 Gas 利用率，
$\tilde{P}$ 表示打包比例，$\tilde{R}_{safe}$ 和 $\tilde{E}_{safe}$
分别表示风险暴露安全得分和头尾风险安全得分。

风险类指标原始值越低越好，因此本文将其转换为安全得分：

\[
\tilde{R}_{safe}
=
1 -
\min\left(\frac{RiskExposure}{RiskRef}, 1\right),
\]

\[
\tilde{E}_{safe}
=
1 -
\min\left(\frac{Edge10Risk}{EdgeRef}, 1\right).
\]

需要说明的是，$S_{\text{trade}}$ 并不表示客观意义上的全局最优，
而是在本文预设目标偏好下衡量不同排序策略的相对折中表现。
```

### 2.4 建议写入权重

如果代码中使用如下权重：

```python
TRADE_SCORE_WEIGHTS = {
    "fee": 0.25,
    "fairness": 0.20,
    "old_tx": 0.10,
    "gas": 0.10,
    "packing": 0.05,
    "risk_safety": 0.20,
    "edge_safety": 0.10,
}
```

论文中可以写：

```latex
本文采用如下预设权重：
$w_f=0.25$，$w_j=0.20$，$w_o=0.10$，$w_g=0.10$，
$w_p=0.05$，$w_r=0.20$，$w_e=0.10$。
该权重设置使收益、等待公平、系统效率和风险安全均参与评价，
避免单纯偏向某一目标。
```

## 3. 修改项三：明确 `trade_score` 是评价指标还是选模指标

### 3.1 当前问题

当前代码中已经实现了 `trade_score`，但默认验证选模可能仍然使用：

```text
two_stage
```

这本身不是错误，但需要明确：

- 如果 `trade_score` 只是测试阶段的评价指标，则论文不能说模型是按 `trade_score` 直接优化得到的；
- 如果论文要主打“给定折中函数下最优”，则建议使用 `trade_score` 作为验证选模指标。

### 3.2 两种可选方案

#### 方案 A：`trade_score` 作为评价指标

保持当前：

```bash
--val-metric two_stage
```

论文写法：

> 本文采用 two-stage 规则进行模型选择，以保证基本可行性；同时在测试阶段使用 \(S_{\text{trade}}\) 作为辅助综合评价指标，分析不同方法在预设目标偏好下的折中表现。

适合叙事：

```text
约束可行性优先 + 多目标折中分析
```

#### 方案 B：`trade_score` 作为选模指标

正式实验使用：

```bash
--val-metric trade_score
```

论文写法：

> 本文采用 \(S_{\text{trade}}\) 作为验证集选模指标，并在测试集上报告相同指标，以保证模型选择和最终综合评价目标一致。

适合叙事：

```text
给定目标偏好下的多目标折中最优
```

### 3.3 建议选择

如果当前小论文主线是：

> 多目标折中最优

建议优先试一次：

```bash
--val-metric trade_score
```

如果主线是：

> 约束可行性优先

则继续使用：

```bash
--val-metric two_stage
```

但必须在论文中说明清楚。

## 4. 修改项四：降级旧的 `composite_score` 叙事

### 4.1 当前问题

代码中现在存在多个综合指标：

```text
composite_score
constrained_fee_score
risk_adjusted_fee_score
trade_score
risk_aware_trade_score
constrained_trade_score
two_stage_selection_score
```

如果论文中全部主打，会造成叙事混乱。

审稿人可能会问：

> 你到底用哪个综合指标判断“折中最优”？

### 4.2 建议分层

| 层级 | 指标 | 论文位置 |
|---|---|---|
| 原始主指标 | `block_fee`, `fairness`, `risk_exposure`, `edge10_risk`, `gas_util`, `packing_ratio` | 主表 |
| 主综合指标 | `trade_score` | 正文辅助表 |
| 风险增强分析 | `risk_aware_trade_score` | 附表/补充分析 |
| 约束分析 | `constrained_trade_score`, `constrained_fee_score` | 约束结果表 |
| 旧综合指标 | `composite_score` | 降级为诊断，不建议主打 |

### 4.3 论文推荐表述

> 正文以原始指标和 \(S_{\text{trade}}\) 为主要评价依据；风险增强得分和约束型得分作为补充分析，用于观察不同目标偏好下策略的 operating point 变化。

---

## 5. 修改项五：规范 `RiskRef` 和 `EdgeRef` 的来源

### 5.1 当前问题

风险安全得分需要参考值：

```text
RiskRef
EdgeRef
```

如果参考值来源不清楚，审稿人可能质疑：

> 这个安全分是不是看完结果后设定的？

### 5.2 建议口径

`RiskRef` 和 `EdgeRef` 应来自：

1. 验证集中的 Gas Priority 风险水平；
2. 验证集所有方法中的最大风险值；
3. 实验协议预设阈值。

不建议来自：

```text
测试集结果
```

### 5.3 建议代码命名

如果当前有：

```python
TRADE_SCORE_EDGE_REF = VALIDATION_TOP10_RISK_CEIL
```

但实际用于 `edge10_risk`，建议改成：

```python
VALIDATION_EDGE10_RISK_CEIL = 0.30
TRADE_SCORE_EDGE_REF = VALIDATION_EDGE10_RISK_CEIL
```

这样语义更清楚。

### 5.4 论文推荐表述

> 风险安全得分中的参考值基于验证集或实验协议预设阈值确定，测试集不参与参考值选择，以避免测试信息泄漏。

## 6. 修改项六：正式实验前做单 seed sanity check

### 6.1 为什么需要 sanity check

加入 `trade_score` 后，正式 5 seed 前应先确认：

- `trade_score` 是否正常计算；
- `risk_aware_trade_score` 是否正常计算；
- `constrained_trade_score` 是否正常计算；
- `table_tradeoff_main.tex` 是否正常生成；
- narrative guard 是否能识别 `trade_score` 相关 winner；
- `edge10_risk` 是否进入主风险分析。

### 6.2 推荐检查命令

示例：

```bash
PYTHONPATH=experiments python3 experiments/run_experiments.py \
  --stages main \
  --seeds 42 \
  --episodes 3000 \
  --eval-episodes 1000 \
  --pool-size 300 \
  --workers 1 \
  --device cuda:0 \
  --operating-mode balanced \
  --val-metric trade_score \
  --output results_single_seed_trade_score_check
```

如果你选择继续使用 two-stage 选模，则改为：

```bash
--val-metric two_stage
```

### 6.3 检查文件

重点检查：

```text
aggregated_main.json
table_main.tex
table_tradeoff_main.tex
narrative_guard.json
v5_protocol_manifest.json
outputs_manifest.json
checkpoint_meta.json
```

### 6.4 判断标准

可以进入正式 5 seed 的条件：

- `outputs_manifest.json` 中 `report_incomplete=false`；
- `table_tradeoff_main.tex` 正常生成；
- `trade_score_mean`、`risk_aware_trade_score_mean`、`constrained_trade_score_mean` 均存在；
- `narrative_guard.json` 不再只围绕 `composite_score`；
- 风险分析中包含 `edge10_risk`；
- 没有出现明显异常值或 NaN。

## 7. 修改项七：论文结论口径同步调整

### 7.1 如果本文方法在 `trade_score` 上第一

可以写：

> 在预设多目标折中得分 \(S_{\text{trade}}\) 下，本文方法取得最优综合表现，说明 PPO 能够学习到符合本文目标偏好的有效交易排序策略。

### 7.2 如果本文方法在 `trade_score` 上不是第一

可以写：

> 本文方法在手续费收益、打包效率和长等待交易服务方面具有优势，并在 \(S_{\text{trade}}\) 下取得有竞争力的折中表现。强启发式方法在特定风险或约束指标上仍具有优势，说明不同排序方法对应不同 operating point。

### 7.3 不能写

无论结果如何，都不要写：

- 本文方法实现全局最优；
- 本文方法所有指标最优；
- 本文方法证明真实 MEV 防护有效；
- 本文方法完全优于所有强基线。

## 8. 最终修改优先级

### P0：正式实验前必须完成

1. 论文中加入 \(S_{\text{trade}}\) 公式和说明；
2. 明确 `trade_score` 是评价指标还是选模指标；
3. 统一正式风险口径为 `edge10_risk`；
4. 将 `composite_score` 降级，不再作为主综合指标；
5. 单 seed sanity check 确认 tradeoff 表能生成。

### P1：强烈建议完成

1. 增加 `VALIDATION_EDGE10_RISK_CEIL`；
2. 让 narrative guard 输出 `trade_score` winner；
3. 让 `table_tradeoff_main.tex` 成为正文或附录表；
4. 在论文中说明 `RiskRef` 和 `EdgeRef` 来源。

### P2：可选增强

1. 增加不同权重下的 sensitivity analysis；
2. 增加 Pareto frontier 图；
3. 增加 `risk_aware_trade_score` 专项分析；
4. 增加 `constrained_trade_score` 与毕业论文约束强化学习部分的衔接说明。

## 9. 最终建议

当前综合折中函数设计方向正确，不建议再扩展新功能。  
正式 5 seed 前，只需要完成以下最小闭环：

```text
统一 edge10_risk 口径
↓
论文加入 S_trade 公式
↓
明确 trade_score 是否用于选模
↓
降低 composite_score 叙事权重
↓
单 seed sanity check
↓
正式 5 seed
```

最终论文主张应调整为：

> 本文不追求客观全局最优，而是在预设收益、公平、风险和效率偏好下，学习一个具有较好综合表现的交易排序 operating point。
