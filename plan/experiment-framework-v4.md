# 第四版实验方案（V4）

## 1. 方案定位

第四版实验方案不是继续增加实验种类，而是在第三版基础上，结合当前代码已经完成的工程修复，进一步把“**实验公平性、运行稳定性、统计有效性、论文可汇报性**”四个维度统一起来。

V4 的核心背景是：当前实验代码已经补齐了以下关键能力：

- 主实验与基线方法可基于**共享候选交易池**进行公平比较；
- 奖励消融与结构消融现已复用同一 seed 下的**共享评估池**；
- 一键脚本可分别输出**风险比例 / 候选池规模 / 费率倍率**三类鲁棒性结果；
- `No-ActionMask` 结构消融不会再因非法动作循环而长时间卡住；
- `pool_size` 已有显式上界校验，不再允许超过环境实现支持范围；
- 显著性检验仅在依赖存在且 seed 数量足够时才运行，避免输出“形式合法、统计无意义”的结果。

因此，V4 的目标不再是“让实验能跑”，而是让实验结果能够更稳地支撑论文中的核心论断。

---

## 2. V4 的总目标

第四版实验方案聚焦以下三个目标：

### 2.1 目标一：保证主实验比较公平

所有方法必须在**完全相同的一批 episode 交易池**上比较，避免因样本差异导致收益、公平性或风险暴露差异被误解释为“方法优劣”。

### 2.2 目标二：保证批量实验稳定结束

实验脚本在默认配置、消融配置和鲁棒性配置下都应能稳定完成，不因非法动作循环、输入越界或统计依赖缺失而在中途失败。

### 2.3 目标三：保证论文汇报结果可直接落表

一键编排的输出文件结构应与正文表格结构对应：主实验一套，三类鲁棒性各一套，奖励消融一套，结构消融一套，显著性结果一套（在满足条件时生成）。

---

## 3. 第四版研究问题

V4 仍保留 V3 的四个研究问题，但对每个问题增加更明确的“证据要求”。

### RQ1 主实验有效性

在统一主场景下，本文方法是否相较于 FIFO、Gas Priority、Heuristic Risk-Aware、Fee-Risk Linear、Fair-Fee Greedy 实现更优综合折中。

**V4 证据要求：**

- 所有方法使用同一批共享评估池；
- 主表给出跨 seed 的均值 ± 标准差；
- 至少报告收益、公平性、风险暴露、Gas 利用率四项主指标；
- 若 seed 数不少于 2，则补充显著性结果。

### RQ2 泛化与鲁棒性

当风险比例、候选池规模、风险交易费率倍率变化时，方法性能是否保持稳定。

**V4 证据要求：**

- 三类鲁棒性实验独立输出，不允许再混成单一结果文件；
- 每一类鲁棒性都应能生成对应 JSON 与 LaTeX 表格；
- 三类实验都使用共享评估池保证方法间公平比较。

### RQ3 组件必要性

奖励项和结构组件是否对最终性能有独立贡献。

**V4 证据要求：**

- 奖励消融与结构消融都必须能稳定结束；
- 同一 seed 下所有消融变体复用同一批共享评估池；
- `No-ActionMask` 不再允许出现无限循环；
- 结构消融至少报告 `block_fee / fairness / risk_exposure / gas_util` 四项指标。

### RQ4 行为可解释性

本文方法是否真的学会了“中部安置高风险交易、同时保留高收益与高等待交易”的排序行为，而不是简单套用一个线性规则。

**V4 证据要求：**

- 至少保留一组典型区块排序案例分析；
- 使用 `risky_rank`、`top10_risk`、`late_promo` 辅助解释策略行为；
- 如时间允许，补一组与线性评分基线的行为对照。

## 4. 默认实验场景（V4 基准）

V4 继续沿用 V3 主场景，不再改动默认任务设定：

| 参数 | 设置 |
|------|------|
| 候选池大小 `N` | 300 |
| 区块 Gas 上限 | 30,000,000 |
| 风险交易比例 `p_risk` | 15% |
| 风险交易费率倍率 | 2.0 |
| 独立随机种子 | 5 个 |
| 每个种子训练回合 | 3,000 |
| 每个种子评估回合 | 1,000 |

### 4.1 pool_size 取值规范

V4 明确规定：

- `pool_size` 的合法范围为 `[50, 500]`；
- 正式实验只使用 `{100, 200, 300, 500}` 这四个鲁棒性规模；
- 不允许使用超过 500 的池规模做正式结果，因为当前环境的 padded 特征矩阵上限就是 500。

### 4.2 seed 取值规范

- 正式实验使用 5 个 seed；
- 小规模 dry-run 可用 1 个 seed；
- 但 **1 个 seed 的结果不得写“显著优于/显著劣于”**，因为统计意义不足。

## 5. 方法与基线设置

### 5.1 本文方法

本文方法继续采用 PPO 逐笔选择交易，保留：

- 候选交易特征输入；
- 区块状态输入；
- STOP 动作；
- 序列摘要；
- 合法动作掩码。

### 5.2 基线方法

V4 固定保留如下 5 个基线：

1. FIFO
2. Gas Priority
3. Heuristic Risk-Aware
4. Fee-Risk Linear
5. Fair-Fee Greedy

### 5.3 V4 对公平比较的强制要求

第四版明确规定：**RL 与所有基线必须在同一批共享候选池上评估**。

这条要求不再只是论文层面的描述，而是实验协议的一部分。若后续改代码时破坏这一点，视为严重实验设计回退。

### 5.4 建议新增基线（可选）

如时间允许，仍建议补一个更强启发式基线：

- **Center-Aware Greedy**：基于收益排序，同时显式惩罚高风险交易出现在头尾位置。

V4 认为这个基线的主要作用不是提升实验数量，而是提升审稿抗质疑能力。

## 6. 奖励函数与环境规则（V4 约束）

### 6.1 奖励函数保持不变

V4 继续沿用当前版本奖励函数：

\[
r_t = \alpha r_{fee}(t) + \beta \Delta r_{fair}(t) - \gamma r_{risk}(t) + \eta r_{valid}(t)
\]

其中：

- `r_fee(t)`：归一化手续费收益；
- `Δr_fair(t)`：增量 Jain 公平性；
- `r_risk(t)`：高风险交易位置惩罚；
- `r_valid(t)`：无效动作 / 提前停止惩罚。

### 6.2 V4 对非法动作处理的规定

由于结构消融中存在 `No-ActionMask` 配置，V4 明确规定：

- 非法动作不能造成无限循环；
- 若策略在无掩码设置下持续输出非法动作，应尽快惩罚并终止 episode；
- 结构消融的目标是检验“去掉掩码后性能如何变化”，而不是检验“脚本能否被卡死”。

### 6.3 V4 对 STOP 的规定

- STOP 仍保留；
- 在打包比例较低时继续限制过早 STOP；
- 若策略通过 STOP 回避决策，应由奖励项进行抑制。

## 7. 评价指标（V4 保留）

### 7.1 主指标

1. 区块手续费收益 `block_fee`
2. 等待时间均衡性 `fairness`
3. 风险暴露度 `risk_exposure`
4. Gas 利用率 `gas_util`
5. 风险交易平均相对位置 `risky_rank`
6. 打包比例 `packing_ratio`

### 7.2 辅助指标

7. 区块前 10% 位置高风险交易占比 `top10_risk`
8. 后到达高手续费交易被提前打包比例 `late_promo`

### 7.3 V4 指标使用原则

- 主实验正文最少使用前 4 项主指标；
- `risky_rank`、`top10_risk` 用于解释“高风险交易是否被推离头部/尾部”；
- `late_promo` 用于解释“收益驱动是否压过公平驱动”。

## 8. 实验矩阵（V4 正式版）

### 8.1 主实验

- 场景：`N=300, p_risk=15%`
- 比较对象：本文方法 + 5 个基线
- 输出：
  - 主实验 JSON
  - 主实验 LaTeX 表格
  - 如满足条件，则输出显著性 JSON 与 LaTeX

### 8.2 鲁棒性实验 A：风险比例

- `p_risk ∈ {5%, 10%, 15%, 20%, 30%}`
- 固定 `N=300`
- 输出独立文件：
  - `aggregated_robustness_risk.json`
  - `table3_risk_content.tex`

### 8.3 鲁棒性实验 B：候选池规模

- `N ∈ {100, 200, 300, 500}`
- 固定 `p_risk=15%`
- 输出独立文件：
  - `aggregated_robustness_pool.json`
  - `table3_pool_content.tex`

### 8.4 鲁棒性实验 C：风险交易费率倍率

- `multiplier ∈ {1.2, 1.5, 2.0, 3.0}`
- 固定 `N=300, p_risk=15%`
- 输出独立文件：
  - `aggregated_robustness_fee.json`
  - `table3_fee_content.tex`

### 8.5 奖励消融

1. FeeOnly
2. Fee+Fair
3. Fee+Risk
4. Full Reward

输出：

- `ablation_reward.json`
- `table_ablation_reward.tex`
- `ablation/shared_pools/seed_*.json`

### 8.6 结构消融

1. No-SeqSummary
2. No-ActionMask
3. No-STOP
4. Full Model

输出：

- `ablation_struct.json`
- `table_ablation_struct.tex`
- `ablation/shared_pools/seed_*.json`

### 8.7 正式评估 checkpoint 规则

- 训练过程同时保存 `best_model.pt` 与 `final_model.pt`；
- 所有正式评估统一加载 `best_model.pt`；
- `best_model.pt` 的定义为训练过程中单回合累计奖励最高时对应的模型参数。

## 9. 统计报告规范（V4）

### 9.1 显著性检验运行条件

显著性检验只在满足以下两个条件时运行：

1. 已安装 `scipy`；
2. 参与聚合的 seed 数量不少于 2。

若任一条件不满足：

- 允许跳过显著性检验；
- 但应在日志中明确说明“因依赖缺失”或“因 seed 数不足”而跳过；
- 正文不得使用“显著优于”之类措辞。

### 9.2 正式统计汇报要求

正式论文版建议：

- 主表使用跨 seed 均值 ± 标准差；
- 当前实现默认采用 Welch t 检验报告关键指标的 p 值；
- 如版面允许，报告 Cohen's d 效应量；
- 补充 95% 置信区间；
- 若显著性表进正文过长，可放附录。

## 10. V4 推荐运行流程

### 10.1 阶段一：预跑（Dry-run）

目的：检查流程，不检查结论。

建议配置：

- `--seeds 42`
- `--episodes 20~50`
- `--eval-episodes 10~20`
- `--pool-size 100`
- `--workers 1`

检查项：

- 主实验结果文件是否生成；
- 三类鲁棒性文件是否都生成；
- 奖励/结构消融是否能正常结束；
- 无显著性条件时是否正确跳过；
- LaTeX 入口脚本是否能读到新文件。

### 10.2 阶段二：小规模正式预验

目的：观察趋势。

建议配置：

- `--seeds 42 123`
- `--episodes 500~1000`
- `--eval-episodes 100~200`

检查项：

- RL 是否明显优于随机/异常策略；
- `No-ActionMask` 是否明显退化；
- 三类鲁棒性趋势是否合理；
- 表格排序与高亮是否正确。

### 10.3 阶段三：正式实验

建议配置：

- `--seeds 42 123 456 789 2025`
- `--episodes 3000`
- `--eval-episodes 1000`
- `--pool-size 300`

正式实验产物用于论文正文与附录。

## 11. V4 与论文写作同步建议

### 11.1 主实验结论写法

只有在满足以下三个条件后，正文才能写结果性结论：

1. 主实验 5 seed 跑完；
2. 共享评估池机制确认无误；
3. 统计条件满足时显著性结果可用。

### 11.2 关于单 seed 结果的表述

单 seed 或 dry-run 结果只能写为：

- “用于验证流程”；
- “用于观察趋势”；
- “不构成正式统计结论”。

### 11.3 关于结构消融的写法

如果 `No-ActionMask` 性能显著下降，应解释为：

- 动作掩码有助于减少非法动作带来的无效探索；
- 同时说明当前实现已避免结构消融因非法动作而卡住，因此性能差异反映的是方法差异，而不是脚本故障。

## 12. V4 最终落地要求

在第四版方案下，实验框架应满足以下验收标准：

1. 主实验比较公平；
2. 三类鲁棒性独立输出；
3. 奖励消融与结构消融稳定结束；
4. 非法输入被提前拒绝；
5. 单 seed 不产生伪显著性结论；
6. LaTeX 输出文件名与脚本入口一致；
7. 正文表格顺序、代码配置、输出文件命名三者保持一致。

若以上 7 条全部满足，则 V4 可以作为论文送审前的正式实验方案。

## 13. 2026-06-30 V5 迁移说明

本文件保留为 V4 历史方案，不再作为当前正式实验协议。当前正式口径以 `plan/第五版实验方案.md` 为准。主要迁移点如下：

1. 奖励函数由旧四项表述迁移为 `step + terminal + valid` 分层奖励；
2. 基线由 5 类基础基线扩展为 7 类，新增 `Center-Insertion Heuristic` 和 `Dynamic Tri-Objective Greedy`；
3. 奖励消融由 `FeeOnly / Fee+Fair / Fee+Risk / Full Reward` 迁移为 `AgeOnly / Age+Risk / Age+TerminalFair / FullBalanced`；
4. checkpoint 默认选模由旧综合或 hypervolume 口径迁移为 `two_stage`；
5. 正式统计由 episode 级或 Welch 口径迁移为 seed 级 paired test + bootstrap CI + Holm 校正；
6. No-ActionMask 结构消融新增非法动作率、非法截断率和连续非法动作诊断；
7. 评估新增平均、P95 和最大推理时间输出。

引用实验方案时，应引用 V5 文档；引用本文件时需明确其为历史方案。
