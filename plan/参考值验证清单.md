# 参考值验证清单

## 目的
验证论文中声称的 RiskRef 和 EdgeRef 参考值（0.30）是否与验证集上 Gas Priority 的实际风险水平一致。

## 论文当前表述

**实验章节（05-experiments.tex）**：
```latex
参考值基于实验协议设定为 RiskRef = 0.30 和 EdgeRef = 0.30。
该设定参考了验证集中 Gas Priority 基线的典型风险水平
（实测范围约 0.28--0.32），使风险安全得分能够反映相对于
主流贪心策略的改进程度。
```

## 验证步骤

### 1. 单 seed 实验完成后

运行命令：
```bash
cd results_single_seed_edge10_final
```

### 2. 检查验证集评估结果

查看文件：
```bash
cat aggregated_main.json | jq '.["Gas Priority"]'
```

或者：
```bash
grep -A 20 "Gas Priority" aggregated_main.json
```

### 3. 提取关键指标

需要检查的指标：
- `risk_exposure_mean`：总体位置敏感风险暴露
- `edge10_risk_mean`：头尾 10% 敏感区风险

### 4. 验证范围

**预期结果**：
- `risk_exposure_mean` 应该在 **0.25 - 0.35** 范围内（理想：0.28 - 0.32）
- `edge10_risk_mean` 应该在 **0.25 - 0.35** 范围内（理想：0.28 - 0.32）

**如果在范围内**：
- ✅ 论文表述准确，无需修改
- 在正式实验报告中提及实测值作为验证

**如果超出范围**：
- 需要调整论文表述或代码参考值
- 参见下方"应急方案"

## 可能的结果与应对

### 情况 A：完全符合预期（0.28 - 0.32）

**行动**：
- ✅ 保持论文当前表述
- 在论文中可以更新为精确值，如"实测为 0.29 和 0.31"

**论文可选更新**：
```latex
参考值基于实验协议设定为 RiskRef = 0.30 和 EdgeRef = 0.30。
该设定与验证集中 Gas Priority 基线的实际风险水平高度一致
（实测分别为 0.29 和 0.31），使风险安全得分能够反映...
```

### 情况 B：略有偏差（0.25 - 0.35，但不在 0.28 - 0.32）

**示例**：risk_exposure = 0.26, edge10_risk = 0.34

**行动**：
- 调整论文表述，扩大范围描述
- 说明参考值是"接近"而非"精确等于"

**论文修改**：
```latex
参考值基于实验协议设定为 RiskRef = 0.30 和 EdgeRef = 0.30。
该设定接近验证集中 Gas Priority 基线的实际风险水平
（实测 risk_exposure 约 0.26，edge10_risk 约 0.34），
使风险安全得分能够反映相对于主流贪心策略的改进程度。
```

### 情况 C：显著偏差（< 0.25 或 > 0.35）

**示例**：risk_exposure = 0.15, edge10_risk = 0.18（风险远低于预期）

**可能原因**：
1. Gas Priority 本身风险就不高（在当前仿真设定下）
2. 风险交易定义或生成参数需要调整
3. 0.30 的阈值设定过于宽松

**行动选项**：

**选项 1：调整代码参考值**（推荐，如果时间允许）
```python
# config.py
VALIDATION_RISK_CEIL = 0.20  # 根据实测调整
VALIDATION_EDGE10_RISK_CEIL = 0.20

TRADE_SCORE_RISK_REF = VALIDATION_RISK_CEIL
TRADE_SCORE_EDGE_REF = VALIDATION_EDGE10_RISK_CEIL
```

重新运行实验。

**选项 2：调整论文表述**（应急方案）
```latex
参考值基于实验协议设定为 RiskRef = 0.30 和 EdgeRef = 0.30，
表示风险暴露的可接受上限。该阈值高于验证集中 Gas Priority 
基线的实际风险水平（实测约 0.15 和 0.18），使得风险安全得分
能够容纳一定的风险改进空间。测试集不参与参考值选择。
```

**选项 3：改用 Gas Priority 实测值**（更科学，但需改代码）

修改 config.py：
```python
# 需要在实验运行时动态设置
# 在 run_experiments.py 中：
gas_priority_risk = evaluate_gas_priority_on_validation()
TRADE_SCORE_RISK_REF = gas_priority_risk["risk_exposure_mean"]
TRADE_SCORE_EDGE_REF = gas_priority_risk["edge10_risk_mean"]
```

论文修改：
```latex
参考值 RiskRef 和 EdgeRef 根据验证集中 Gas Priority 基线的
实际风险水平动态确定（分别为 0.15 和 0.18），
使风险安全得分直接反映相对于贪心基线的改进程度。
```

### 情况 D：风险值过高（> 0.40）

**示例**：risk_exposure = 0.45, edge10_risk = 0.50

**问题**：Gas Priority 风险比预期高很多，可能意味着：
1. 风险交易生成参数过高
2. 风险定义过于严格
3. Gas Priority 确实在这个场景下风险很高

**行动**：
- 先确认这是否合理（检查其他基线的风险）
- 如果所有方法风险都高，可能是场景设定问题
- 考虑调高参考值或调整风险生成参数

## 验证脚本

可以使用以下 Python 脚本快速检查：

```python
import json

# 读取实验结果
with open("results_single_seed_edge10_final/aggregated_main.json") as f:
    data = json.load(f)

# 提取 Gas Priority 的风险指标
gp = data.get("Gas Priority", {})
risk_exp = gp.get("risk_exposure_mean", None)
edge10 = gp.get("edge10_risk_mean", None)

print(f"Gas Priority 验证集风险水平：")
print(f"  risk_exposure_mean: {risk_exp:.4f}" if risk_exp else "  risk_exposure_mean: N/A")
print(f"  edge10_risk_mean: {edge10:.4f}" if edge10 else "  edge10_risk_mean: N/A")

# 判断是否在预期范围
if risk_exp and edge10:
    risk_ok = 0.25 <= risk_exp <= 0.35
    edge_ok = 0.25 <= edge10 <= 0.35
    
    print(f"\n参考值验证：")
    print(f"  RiskRef = 0.30 {'✅ 合理' if risk_ok else '⚠️ 需要调整'} (实测 {risk_exp:.4f})")
    print(f"  EdgeRef = 0.30 {'✅ 合理' if edge_ok else '⚠️ 需要调整'} (实测 {edge10:.4f})")
    
    if risk_ok and edge_ok:
        print("\n✅ 论文表述准确，无需修改")
    else:
        print("\n⚠️ 建议调整论文表述或代码参考值")
else:
    print("\n❌ 数据不完整，请检查实验输出")
```

保存为 `scripts/verify_risk_refs.py`，运行：
```bash
cd /home/yexianglun/my_projects/LUNWEN
PYTHONPATH=. python scripts/verify_risk_refs.py
```
